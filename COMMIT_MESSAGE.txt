Introduce intelligent content analysis capabilities, transitioning Spectra
from pure topology (size/location) to typography (meaning/risk). This release
establishes a tiered inspection architecture that maintains zero-latency
performance while adding semantic understanding.

## Core Features

### Analysis Module (cli/src/analysis/)
- **entropy.rs**: Shannon entropy calculation on file headers (8KB sampling)
  - Detects encrypted, compressed, or obfuscated files
  - Returns 0.0 (uniform) to 8.0 (random) scale
  - Microsecond-level performance

- **heuristics.rs**: Pattern-based risk detection
  - 5-level classification: None, Low, Medium, High, Critical
  - Detects sensitive files: passwords, keys, certificates, SSH keys, wallets
  - Path-aware matching (e.g., .ssh/id_rsa â†’ Critical)
  - Zero file reads - filename/path analysis only

- **semantic.rs**: Optional AI content classification
  - rust-bert DistilBERT zero-shot classification
  - Categories: legal, financial, technical, etc.
  - Feature-gated to keep base binary small
  - Only processes low-entropy (text) files

## Architecture

### Tiered Inspection Model
- Tier 0: Metadata (size, path) - nanoseconds
- Tier 1: Heuristics (entropy, patterns) - microseconds
- Tier 2: Semantic (AI) - milliseconds, optional

### Feature Flags
- `default`: Base scanning engine (~2-5MB binary)
- `semantic`: Adds rust-bert (~500MB with LibTorch)

### Performance
- Post-scan analysis on top N files only (--limit configurable)
- Analysis overhead: ~1-2ms per file (Tier 1), ~10-50ms (Tier 2)
- Privacy-first: All processing local, read-only first 8KB
